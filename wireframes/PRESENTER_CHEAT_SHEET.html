Q&A Bank — Technical Questions Likely to Come to You
Portal vs Zoho CRM — same system?
Connected but separate. Zoho CRM is the system of record. The applicant portal is a custom app on Zoho Catalyst — talks to CRM via API and writes everything back. External users never touch CRM directly.
MFA and user security?
Zoho CRM supports MFA natively — authenticator apps and security keys. Admins can enforce it globally for specific roles. Account recovery process doesn't create a back door.
Accidentally delete 500 records?
Zoho has a recycle bin — deleted records sit there for a configurable period before permanent removal. Mass restoration brings records back with all associated history and attachments intact.
GDPR right-to-erasure?
We configure what erasure means in your context — which fields are deleted, which are anonymised, what's retained for legal obligation. A request triggers a workflow. The audit trail of the erasure itself is retained.
Restrict access to specific IP ranges?
Yes — Zoho supports IP allowlisting at organisation level. Restrict access so only authorised networks can connect. Configuration setting, not a custom build.
Documents encrypted at rest?
Zoho uses AES-256 at rest and TLS 1.2+ in transit. Documents uploaded through the portal stored within Zoho's infrastructure with those standards.
Reapply in future cycles?
Yes — the account persists within a retention window we configure. They return, start a new application, and their history is still there.
Arabic / right-to-left languages?
RTL support is confirmed in scope for Phase 1. We haven't shown it today because it requires additional configuration — we'd rather be accurate than approximate.
Rejected applicant — what happens?
Retained in full. Blueprint stage moves to Declined, notification to applicant, every stage/score/decision stays in the system. Nothing deleted. Fully auditable.
Blind review — how does it actually work?
Each reviewer's assessment is a separate record. Field-level security means a reviewer can only read their own. When both submit, a workflow flips All Reviews Complete, which unlocks mutual visibility. Blueprint trigger, not manual.
Safeguarding escalation beyond NC?
IO Super Admin is notified at the same moment as the NC Admin. The record stays locked until IO-level access explicitly resolves the flag. Full audit trail retained.
If Zoho changes something that breaks our config?
Zoho is a mature platform with stable APIs and transparent upgrade path. We monitor platform changes as part of ongoing support. Hand to Paul for the support-arrangement specifics.
