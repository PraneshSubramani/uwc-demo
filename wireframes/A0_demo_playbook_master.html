How a tester should use this document

This is a functional UAT test plan for verifying the platform the way a real customer uses it — organised by user role, not by demo scenario. Each role has a complete end-to-end suite of test cases covering happy paths, edge cases, negative cases, permission boundaries, Blueprint gates, automations and data validation. A final Engine suite tests the Blueprint state machine and every automation in isolation. Execute top to bottom; many suites share state (e.g. a reviewer submits before the blind-visibility check).
