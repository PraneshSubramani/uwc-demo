What you're testing

Scenario E is governance health. Members sit on Boards with time-bounded terms. When a term expires the membership auto-lapses, and any competency that member held drops out of coverage — turning a cell red in the Competency Matrix. The centrepiece is that the matrix surfaces 4 board-level gaps caused by two expired terms plus two never-covered regions.

The mechanism to test: Term_Expires (formula) → Membership_Status = Lapsed (scheduled function) → Gap_Count (scheduled function) → red cells in the widget.

Field / config names are from the build spec — confirm against the live org

Unlike Scenario D, the governance module field names have not yet been verified against the live EU org. Names tagged confirm should be checked before relying on them (e.g. via GET /crm/v8/settings/fields?module=Governance_Memberships).

Manual build — not API-creatable

The formula field, scheduled functions, WR-E-1/E-2 rules, list-view conditional formatting (red EXPIRED badge), and the HTML widget all need building in Setup. They can't be created/verified via the CRM REST API.

The cause → effect chain
Member term expires ──▶ Membership_Status = Lapsed (lapse_expired_memberships) ──▶ that member's competencies drop out of active coverage ──▶ Gap_Count++ for those competencies (recompute_competency_gaps) ──▶ red cell in the Competency Matrix widget

Pre-test setup (do once)

1. Confirm the 5 seed bodies exist (UWC International Board + Brazil/Kenya NC Boards + Safeguarding Advisory + Finance Sub-Committee) and the 5 seed memberships, two of which are expired: Dr Ravi Menon (term to 31/03/2025) and Thomas Berger (to 31/12/2024).
2. Pre-seed Gap_Count manually — the recompute runs only overnight; without seeding, the matrix shows zero gaps on stage.
3. Confirm Competencies with Required_Board = true include Marketing & Comms, Asia-Pacific Regional, Europe Regional, Americas Regional.

Tests

1 Governance Bodies list renders IO Super Admin

Do
Open the Governance Bodies list view.
Pass
5 bodies with status badges and member counts. (Create "Programme Quality Committee" via + New → it appears with Member_Count = 0.)

2 Term_Expires formula computes IO Super Admin

Do
On any membership, set Appointed_Date = 01/01/2023, Term_Length = 3 → Save.
Pass
Term_Expires computes to 31/12/2025 (Appointed + Term_Length × 365).
If wrong
The formula field is mis-defined or Term_Length is empty/non-integer.

3 Expired memberships show red + are Lapsed IO Super Admin

Do
Open UWC International Board → Members related list. Then open the Historical Memberships saved view (Membership_Status != Active).
Pass
Members list shows 5, with two red EXPIRED badges (Dr Ravi Menon, Thomas Berger). Historical view lists exactly those two.
If they're not Lapsed
lapse_expired_memberships hasn't run. Run it manually (Functions → Save & Execute), or the badge formatting / saved-view filter is wrong. Note: Zoho can't auto-flip status on a date alone — it needs the scheduled function.

4 Add a member IO Super Admin

Do
On UWC International Board → + Add Member → Contact = Akanksha Anand, Role = Advisor, Appointed = today, Term Length = 3, Competencies = [Programme Design, Youth Engagement] → Save.
Pass
A new Active membership appears; the board's Member_Count increments.

5 Competency Matrix shows 4 red cells IO Super Admin · CENTREPIECE

Do
Open the Competency Matrix dashboard (the HTML widget).
Pass
Grid renders rows = boards, cols = required competencies, with 4 red cells: Marketing & Comms Asia-Pacific Regional Europe Regional Americas Regional. Footer shows Gap_Count per competency. Loads in < 2 seconds.
Trace the why
Marketing & Comms is red because Thomas Berger (its only holder) lapsed; Asia-Pacific because Dr Ravi Menon lapsed; Europe + Americas because no member ever held them. If a cell is wrongly green, that member's Membership_Status is still Active, or Gap_Count wasn't recomputed.
If all cells green / zero gaps
recompute_competency_gaps() hasn't run since the lapses — pre-seed Gap_Count or run the function manually.

6 Network Summary report IO Super Admin

Do
Open the Network Summary report/dashboard.
Pass
Pie of body types (Board / Committee / Advisory), bar of competency coverage %, and a count of lapsed memberships network-wide (includes Berger + Menon). Drill-through on a tile opens the underlying records.

7 Renewal alert is wired (WR-E-1) IO Super Admin

Do
Setup → Workflow Rules → WR-E-1. (Optional functional test: set a membership's Term_Expires = today + 60 and run email_renewal_due.)
Pass
Rule exists and is active; the function emails IO Super Admin + the Body Chair for memberships expiring in 60 days.

Re-running the test

If you manually flipped a membership to test lapsing, reset its Membership_Status back to Active (or re-run demo_reset) and re-seed Gap_Count before the next dry-run, so the matrix shows the intended 4 gaps.
