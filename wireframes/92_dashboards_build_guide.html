IO NC Health Dashboard

The Scenario F centrepiece: the IO sees the whole network at a glance and clicks through to any National Committee. Build it under Reports & Dashboards → Dashboards → + Create Dashboard. Name it IO NC Health Dashboard in the "Untitled Dashboard" box first.

Values below reflect the live EU org (11 NCs). When all 7 components are saved, set sharing to Everyone so every profile can view it.

Choose the "Basic" KPI style

When you click KPI, the picker offers Basic / Standard / Growth Index / Scorecard / Rankings. Pick Basic (top-right) — it shows a single clean number, exactly what the demo needs. Avoid Standard & Growth Index (they add a %-change arrow vs last period, meaningless for operational counts and implying a target you haven't set). Set Duration to All Time so the KPI counts every record regardless of created date.

The measure to select is Count of Records.

Prerequisite: Training Status must be a colour-coded picklist

NC_Volunteers.Training_Status was originally a text field — the EXPIRED-OR-MISSING filter won't work reliably against free text. It was converted to a picklist with four colour-coded values and all 23 volunteer records repopulated: CURRENT #D1FAE5 · EXPIRING SOON #FEF3C7 · EXPIRED #FEE2E2 · MISSING #F3F4F6.

Kenya is the worked example: Grace = CURRENT, Daniel = EXPIRED, Aisha = MISSING (→ the 2 training gaps). Refresh the NC Volunteers related list on the Kenya record to see the chips.

Pre-seed before the live demo

MoU_Health on the NC records and the volunteer Training_Status are written by nightly scheduled functions (recompute_mou_health 03:15 UTC, etc.). Seed/verify these values the evening before so the dashboard renders correctly on stage rather than waiting for the overnight run.
